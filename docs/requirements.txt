✅ IIS + FastAPI (uvicorn) Reverse Proxy 세팅 정리
1. 기본 준비
도메인 연결
domain.com (aipos24.com) → Vultr 서버 IP (45.76.xxx.xxx) A레코드 세팅
nslookup aipos24.com → 올바른 IP 확인
FastAPI 실행 확인
서버 내에서 uvicorn backend.main:app --host 127.0.0.1 --port 8000 실행

http://127.0.0.1:8000
 정상 동작 확인
2. IIS 기본 사이트 생성
IIS Manager → Sites → Add Website
Site name: aipos24.com
Physical Path: C:\inetpub\wwwroot\aipos24
Binding: http / Port 80 / Host name: aipos24.com
(선택) www.aipos24.com 추가 바인딩

3. SSL 인증서 적용 (win-acme 사용)
win-acme (wacs.exe) 실행 → N (새 인증서 생성)
aipos24.com, www.aipos24.com 선택
인증 완료 후 IIS에 자동 바인딩
Site Bindings에서 https / 443 / SSL certificate 선택

4. ARR (Application Request Routing) 설치 & 활성화
Web Platform Installer → ARR 3.0 설치
IIS Manager → Server Root 선택 → Application Request Routing Cache → 오른쪽 → Server Proxy Settings

✅ Enable Proxy 체크
5. URL Rewrite 룰 추가 (Reverse Proxy)
해당 사이트(aipos24.com) 선택 → URL Rewrite → Add Rules
Inbound Rule → Blank Rule
Name: ReverseProxyToUvicorn
Requested URL: Matches the Pattern
Using: Regular Expressions
Pattern: (.*)
Action
Action type: Rewrite
Rewrite URL: http://127.0.0.1:8000/{R:1}
적용 후 사이트 Restart

6. 테스트
http://aipos24.com 접속 → uvicorn 서비스 정상 표시
https://aipos24.com 접속 → 인증서(SSL) 적용된 HTTPS 확인

🚀 핵심 포인트
ARR Proxy 반드시 Enable Proxy 체크
Rewrite URL → 127.0.0.1:8000 으로 연결
SSL 인증서는 win-acme로 자동 갱신됨 (기본 60~90일 마다)
FastAPI는 IIS에서 직접 구동되지 않고 uvicorn → IIS 프록시 구조
